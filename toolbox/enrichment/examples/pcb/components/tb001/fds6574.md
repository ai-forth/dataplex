## Summary

`FDS6574` is one of the highest-value component records to normalize for TB001
because it appears in both the page-4 and page-5 control/power boundary and is
currently the largest blocker to stronger simulation confidence.

## Known From Current Board Evidence

* The part is identified as `FDS6574`.
* The package currently associated with it is `SOIC-8`.
* It appears as `Q1` on page 4.
* It appears as `Q1`, `Q5`, `Q7`, `Q8`, `Q10`, and `Q11` on page 5.
* Current corpus interpretation treats it as part of the higher-risk switching
  family rather than a passive or low-consequence support part.

## Manufacturer Characteristic Status

Current status is intentionally incomplete.

Known:

* MPN string: `FDS6574`
* package evidence in the current TB001 corpus: `SOIC-8`

Still required from manufacturer sources:

* exact manufacturer identity
* verified pinout
* MOSFET polarity and channel arrangement
* threshold and on-resistance characteristics
* current and thermal limits
* availability of vendor SPICE model

## Manufacturer Source Intake

Current primary-source status:

* TB001 migration notes record that a direct current onsemi lookup did not
  recover a live product page or datasheet from the guessed legacy path.
* Because of that, the current catalog should treat all electrical-role claims
  about `FDS6574` as unresolved unless backed by an archived manufacturer
  source.

Current practical rule:

* use the BOM-backed `SOIC-8` package fact
* do **not** treat polarity, pinout, or device strength as manufacturer-proven
  yet
* keep simulation use explicitly labeled as a placeholder model

## Simulation Relevance

This part currently dominates simulation uncertainty in the page-5 switching
cases.

The current comparison work already suggests that a stronger interpretation of
the `FDS6574` family produces a more plausible result than a single weak pass
element, but the manufacturer-grounded behavior is still missing.
